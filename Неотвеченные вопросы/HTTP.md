Протокол HTTP – это способ получения ресурсов из сети.
Данные называются "ресурсами". У каждого ресурса есть свой уникальный адрес URI (Unique Resource Identifier). 

| Адрес                            | Описание                                                       | Пример                                     |
| -------------------------------- | -------------------------------------------------------------- | ------------------------------------------ |
| URI (Unique Resource Identifier) | Уникальный адрес ресурса                                       | `https://librarian.com/books?author=Gogol` |
| URL (Unique Resource Link)       | Адрес (эндпойнт), указывающий на местоположение ресурса в сети | `https://librarian.com/books`              |

Методы HTTP запросов:

| Метод  | Описание                                                                                    | Пример                                          |
| ------ | ------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| GET    | Метод при помощи которого клиентское приложение запрашивает ресурсы у серверного приложения | `GET https://librarian.com/books?author=Gogol`  |
| PUT    | Метод для обновления ресурса целиком                                                        |                                                 |
| POST   |                                                                                             |                                                 |
| PATCH  | Метод для частичного обновления ресурсов (по конкретным полям)                              | `PATCH https://librarian.com/api/v1/books/{id}` |
| DELETE | Метод для удаления конкретного ресурса                                                      |                                                 |

Способы передачи данных **от клиента → серверу** в [[REST API|REST API]]:
1. Через параметры GET-запроса и параметры пути:
   `https://librarian.com/books?author=Gogol` 
   `https://librarian.com/books/{id}
2. Через тело запроса (JSON, XML),
3. Через заголовки (headers),
4. Через cookies.

Способы передачи данных от **сервера → клиенту** в REST API:
1. Через тело ответа (JSON, XML),
2. Через status code,
3. Через заголовки,
4. Через cookies.
  

Статус коды HTTP:

| Код                         | Описание                                                                                | Пример ответа |
| --------------------------- | --------------------------------------------------------------------------------------- | ------------- |
| 200 (OK)                    | OK                                                                                      | <br>          |
| 201 (Created)               | Ресурс успешно создан                                                                   |               |
| 204 (No content)            | Возвращает информацию о том, что удаление прошло успешно и ресурса больше нет           |               |
| 400 (Bad request)           | Переданы некорректные данные или недостаточно данных для создания или получения ресурса |               |
| 401 (Unauthorised)          | Требуется аутентификация для доступа к запрашиваему ресурсу                             |               |
| 403 (Forbidden)             | Ошибка авторизации                                                                      |               |
| 404 (Not found)             | Ресурс не найден                                                                        |               |
| 500 (Internal Server Error) | На сервере произошла ошибка и он не смог успешно обработать запрос                      |               |
|                             |                                                                                         |               |
